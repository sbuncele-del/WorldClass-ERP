## Usage Guide

### After an Incident
1. **Incident Post-Mortem Template** - Document the incident
2. Schedule post-mortem review meeting
3. Track action items to completion

### Incident Severity Reference

| Severity | Definition | Post-Mortem Required |
|----------|------------|:--------------------:|
| P1 - Critical | Service completely unavailable | Yes (within 5 days) |
| P2 - High | Major feature unavailable | Yes (within 7 days) |
| P3 - Medium | Feature degraded | Optional |
| P4 - Low | Minor issue | No |
