01 What We're Hosting

ATG's product is an AI-powered automated forex trading bot. It pulls free market data from Nasdaq.com and Yahoo Finance, uses AI to analyse price action and market conditions, and places trades directly on MetaTrader 5 (MT5).

There are no subscribers, no website, no web dashboard. It is a background process that runs on a server 24 hours a day during market hours, making trades autonomously.

What We Need

ComponentWhat It Does
Windows VPSA virtual server running Windows. MT5 is a Windows application — it needs Windows to run. The AI bot runs on the same server alongside MT5.
MetaTrader 5Installed directly on the Windows VPS. MT5 connects to the forex broker. The AI bot uses MT5's built-in Python API to analyse markets and place trades — no middleman, no bridge service.
Market DataFree — pulled from Nasdaq.com and Yahoo Finance. No paid data feed required.
StorageDisk space for trade logs, AI model data, and price history. 40–100 GB SSD is more than enough.
What we DO NOT need: A Linux server, a CDN, a load balancer, Cloudflare, a managed database, object storage, or multiple servers. This is one bot + MT5 on one Windows VPS. That's it.

How It Works — The Setup

The complete setup on the Windows VPS:

  1. Rent a Windows VPS from a provider (Vultr, Hetzner, or similar)
  2. Install MetaTrader 5 on the VPS and log in with your broker account
  3. Install Python on the VPS
  4. Deploy the AI trading bot — it uses MT5's Python library to read market data, run analysis, and place trades
  5. The bot runs 24/7 during market hours. You can remote-desktop into the VPS anytime to monitor it.
Why Windows VPS?

MetaTrader 5 is a Windows application. It does not run natively on Linux. The simplest, most reliable approach is a Windows VPS where MT5 and the AI bot run together on the same machine. No bridge services, no workarounds, no third-party dependencies between the bot and the broker. One server, one bill.

⚠ Vultr Fine Print — Read Before You Sign Up

USD billing only: Vultr invoices in US Dollars. If the rand moves from R18/$ to R22/$, your R720/mo becomes R880/mo. The service doesn't change — only your bill does. This is real FX risk on a recurring cost.

"Per hour" billing explained: You are charged $0.060/hr for this plan (server + Windows). The monthly price ($40) is a cap — you never pay more than $40/month even though $0.060 × 720 hours = $43.20. The system charges whichever is lower. If you run 24/7 (which you will — the bot trades around the clock), you always pay the capped monthly rate.

Stopping vs. destroying: "Stopping" a server (shutting it down) does NOT stop billing. Your disk, IP address, and resources are still reserved for you. Only destroying/deleting the server stops charges. If you want to pause costs, you must destroy the server — but you lose everything unless you backed it up.

Bandwidth: 3 TB/month included (clearly stated). A trading bot pulling price data uses a fraction of this. Overage charges apply beyond 3 TB but you won't hit it.

IP address: 1 IPv4 included. Additional IPs cost $3/mo each. You only need 1.

Backups: Optional, adds 20% to monthly cost ($8/mo = ~R144/mo). Automatic weekly snapshots. Recommended — if the server dies, you can restore from backup in minutes.

Snapshots: On-demand snapshots are free (up to storage limits). Good for before-and-after backups when updating the bot code.

Windows licence: The $16/mo Windows Server licence is mandatory if you choose a Windows template. You cannot bring your own licence on Vultr cloud compute.

Remote Desktop (RDP): Included with Windows. You connect to your VPS over Remote Desktop to install MT5, deploy the bot, and monitor it. No extra cost.

⚠ Hetzner Fine Print

Bare metal (not cloud): This is a physical machine, not a virtual one. You rent actual hardware. It's powerful, but if it breaks, Hetzner replaces the hardware — which can take hours or longer. During that time, your bot is offline and not trading.

Windows licence: Hetzner can provide Windows Server at an additional monthly fee. Check current pricing — it varies. You install Windows yourself or request it pre-installed.

No cloud flexibility: You cannot "resize" a dedicated server. If you need more RAM or CPU, you order a different server and migrate manually. With cloud providers you click a button.

Full sysadmin responsibility: OS updates, security patches, firewall rules, backups — all on you. There is no managed service layer.

HDD, not SSD: The base RS-8 uses traditional hard drives, not SSDs. HDD is slower for AI model loading and data reads. SSD upgrades are available at extra cost.

Setup fee: Dedicated servers may have a one-off setup fee. Confirm with Hetzner before ordering.

Monthly billing only: No hourly billing. You pay for the full month. If you cancel mid-month, you still owe the full month.

Single point of failure: One physical machine. If it fails, everything stops. No automatic failover. Cloud VPS providers typically recover faster because they can spin up a new VM from a backup.

Pros

  • Cheapest option (~R403/mo incl. VAT)
  • ZAR billing — no exchange rate risk

  • Unmetered bandwidth

What About CloudAfrica?

CloudAfrica (cloudafrica.net) is a South African cloud provider with ZAR pricing. However, they offer Linux only — no Windows VPS option. Since MT5 requires Windows, CloudAfrica is not suitable for this use case.

USD billing: GitHub bills in USD. Same FX risk as Vultr.

CI/CD minutes: If you use GitHub Actions to auto-deploy the bot to your VPS, the 2,000 free minutes/mo is more than enough. You won't hit this limit.

Copilot: GitHub Copilot (AI coding assistant) costs $10/mo extra. Optional, not required.

Recommendation

For a production AI trading bot handling real money: Vultr Windows VPS + GitHub Pro at ~R792/mo.

You get a Windows server in Johannesburg, install MT5, deploy the bot, and trade. Everything is on one machine. Backups are one click. If something breaks, Vultr restores it in minutes. You can resize the server if the bot needs more power.

Hetzner is cheaper (~R475/mo) but it's a physical machine — if it fails, you're offline for hours while real trades could be running. No easy backups, no resizing, more sysadmin work. The saving is not worth the risk when real money is on the line.

The FX risk is real — if the rand weakens to R22/$, Vultr goes to ~R880/mo. But even at R22/$, the reliability advantage is worth it for a live trading bot.
